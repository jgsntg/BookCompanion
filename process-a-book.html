Processing a Book

The end-to-end flow from EPUB/PDF to a queryable book in the app.

1Get the file in place

Put the EPUB (or PDF) somewhere under v0/ — e.g. v0/SomeBook.epub.

2Run extraction (v0)

cd v0
source .venv/bin/activate
python src/extract.py path/to/book.epub
  • Detects fiction vs. nonfiction, then runs LLM extraction chapter by chapter (claims, frameworks, passages, connections, open questions)
  • Writes output/{book-slug}.json
  • Provider/model defaults to Anthropic Claude; pass --provider openai to use GPT instead
  • To re-run specific chapters only: --chapters 1-10 (range), 1,3,5 (list), or both
Cost ballpark: ~$0.30–$0.80 per book (Sonnet, ~25 chapters).

3Ingest into Supabase (v1)

cd v1/scripts
source .venv/bin/activate
python ingest_supabase.py ../../v0/output/{book-slug}.json
  • Reads the extraction JSON, embeds chunks via Voyage (voyage-3-lite), and writes everything to Supabase
  • Requires NEXT_PUBLIC_SUPABASE_URL, SUPABASE_SERVICE_ROLE_KEY, and VOYAGE_API_KEY in v1/scripts/.env
  • Re-ingesting the same book updates it via its dedupe_key — safe to run again after fixing the extraction

4Open it in the app

cd v1/web
npm run dev
  • Visit http://localhost:3000
  • The book appears in your library with status automatically set to Reading
  • Open the book and use the query box to ask questions — retrieval + synthesis is scoped to that book

5Wrap up

  • Mark the book Finished when done (sets finished_at automatically)
  • Add a rating and a short note — notes are embedded too, so they're searchable alongside the extracted content

Logging a book without an EPUB

  1. Go to /library/add
  2. Search Open Library to autofill, or type the title/author manually
  3. Set status, rating, and a one-paragraph note → Add to library

No chapter-level queries are available for these — just the note, embedded as a single searchable chunk.

See v0/README.md and v1/README.md for full setup details.
